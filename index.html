Loop prevention rules — read before building any automation
⚠ These rules caused 12-15x inventory duplication in the old system
Status↔Group loops fired Make.com webhooks repeatedly per order. Every cycle deducted inventory and created a report row. These rules are non-negotiable.
RULE 1 — Status drives group. Group NEVER drives status. ✓ ALLOWED: When Status = Shipped → move item to Shipped group ✗ FORBIDDEN: When item moved to Shipped → set Status = Shipped RULE 2 — ONE trigger creates report rows per board. ✓ ALLOWED: When Status = Shipped → create item in Report board ✗ FORBIDDEN: When Receiving Date changes → create item in Report board ✗ FORBIDDEN: When Shipping Date changes → create item in Report board RULE 3 — ONE webhook per event per board. ✗ FORBIDDEN: Two "When item created" webhooks on the same board RULE 4 — No catch-all status automations. ✗ FORBIDDEN: When Status changes to something → move to group (catch-all) ✗ FORBIDDEN: When Status = something → create a subitem RULE 5 — Idempotency in scenario 1230 is mandatory. Add the data store guard even with clean automations. It's the safety net.

Receiving date + shipping date — why they fail and how to fix
// WHY THEY FAIL Shipping Date problem: "When Status = Shipped → set Shipping Date = today" fires at same time as "When Status = Shipped → create report row" monday.com hasn't written Shipping Date yet when the row is created. Result: blank Shipping Date on report row. Receiving Date problem: Report row created when Status = Shipped. Dustin enters Received Amount days later → Receiving Date set on order board. Nothing syncs it back to the already-created report row. Result: blank Receiving Date on report row forever. // FIX FIX 1 — Shipping Date: In the "Create item in Report" automation, map Shipping Date = today NOT "{Shipping Date column value}" — hardcode it. FIX 2 — Receiving Date: Add automation on Maintenance + Construction boards: TRIGGER: When Received Amount changes AND Receiving Date not empty ACTION: Find report row (match by Request ID) ACTION: Update report Receiving Date = {Receiving Date} ACTION: Update report Received Amount = {Received Amount}
